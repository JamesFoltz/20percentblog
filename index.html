Wolfenstein-Style 3D Renderer

Overview

I’m currently working on a Wolfenstein-style 3D renderer. The basic idea of this project is to make a pseudo-3D world by rendering a 2D map using ray casting. This creates the illusion of depth by drawing vertical lines of varying heights to represent walls, similar to how classic first-person shooters like Wolfenstein 3D achieved their visuals.

How It Works

  1. 2D Grid-Based Map
    • The world is represented as a grid of numbers and symbols, where different values correspond to walls, empty spaces, or objects.
    • this is what a map might look like

      map = [
        "🟥🟥🟥🟥🟥🟥🟥🟥🟥🟥",
        "🟥▫️▫️🚪▫️▫️▫️☠️▫️🟥",
        "🟥▫️🟥🟥🟥🟥▫️▫️▫️🟥",
        "🟥▫️🟥▫️▫️🟥🟥▫️🟥🟥",
        "🟥▫️🟥🟥🚪🟥▫️🟥▫️🟥",
        "🟥▫️▫️▫️▫️🟥▫️🟥🟥🟥",
        "🟥▫️🟥🟥🟥🟥▫️🛡️🟥🟥",
        "🟥▫️▫️☠️▫️▫️▫️▫️🟥🟥",
        "🟥🟥🟥🟥🟥🟥🟥🟥🟥🟥"
    ]
    
    legend = {
        "🟥": "wall",
        "🚪": "door",
        "▫️": "air", 
        "☠️": "enemy",
        "🛡️": "spawn"
    }
  • The player moves within this grid, and their position, along with their viewing angle, determines what they see.

  1. Ray Casting Technique

    • Instead of rendering a full 3D environment, the code shoots out rays from the player’s position in the direction they are looking, one for each vertical column of pixels on the screen.
    • Each ray moves forward until it hits a wall, calculating the distance and angle to determine the correct wall height** and eventualy texture slice.

  2. Rendering the Scene

    • The screen is drawn column by column, with closer walls appearing taller and farther walls appearing shorter to emulate depth.
    • To hide math errors and improve the look, more distant walls are faded out to create a fog effect.

Features I’m Implementing

  • Smooth Player Movement & Rotation
  • Basic Collision Detection
  • Textured Walls

Goals & Challenges

  • The primary goal of this project is to build and understand the math behind 3D to 2D projection.
  • The primary chalenge is the lack of information online about this kind of stuff.

Weak perspective projection

Overview

As part of learning how games render 3d scenes, I have written a weak perspective projection 3d renderer. This means that it uses simple math to approximate 3d to 2d point conversion.

How It Works

  1. Model format
    • Each model is a list of polygons, each made up of lists of 3d points
    • this is what a model might look like
var models = [
  // Cube
  [
    // Front face
    [
      { x: 0, y: 0, z: 0 },{ x: 50, y: 0, z: 0 },{ x: 50, y: 50, z: 0 },{ x: 0, y: 50, z: 0 },
    ],
    // Back face
    [
      { x: 0, y: 0, z: 50 },{ x: 50, y: 0, z: 50 },{ x: 50, y: 50, z: 50 },{ x: 0, y: 50, z: 50 },
    ],
    // Left face
    [
      { x: 0, y: 0, z: 0 },{ x: 0, y: 50, z: 0 },{ x: 0, y: 50, z: 50 },{ x: 0, y: 0, z: 50 },
    ],
    // Right face
    [
      { x: 50, y: 0, z: 0 },{ x: 50, y: 50, z: 0 },{ x: 50, y: 50, z: 50 },{ x: 50, y: 0, z: 50 },
    ],
    // Top face
    [
      { x: 0, y: 50, z: 0 },{ x: 50, y: 50, z: 0 },{ x: 50, y: 50, z: 50 },{ x: 0, y: 50, z: 50 },
    ],
    // Bottom face
    [
      { x: 0, y: 0, z: 0 },{ x: 50, y: 0, z: 0 },{ x: 50, y: 0, z: 50 },{ x: 0, y: 0, z: 50 },
    ]
  ]
];
  1. Point Conversion
    • Weak perspective point conversion works by dividing the x position and y position by the z position. This isn’t very accurate, however, it creates a convincing effect and was used in many early 3d video games such as the early builds of Quake and Doom.

QISH(Quick Inquiry SHell)

QISH is a pseudo-terminal used in place of a typical new tab page for web navigation. As the name sugests, it has many tools to make looking up information more efficient such as duckduckgo style bangs(!gi for google images, !wiki for wikipedia), Quick access to varios apis and a faster, more private search experience.

Why though?

Making this helped me practice the use of web apis as well as various libraries used in the profesional world.

Tools used

  • JQuery, a DOM manipulation javascript library
  • terminal.jquery.js, an addon for JQuery to asist in making web pseudo-terminal aplications
  • keyboard.js, used to add more key interaction utility, allowing for more complex keyboard inputs

Periodic Table Search Tool

I made a website that makes it easy to find and get info on the elements on the periodic table.

Why though?

It helped me to learn more about data parsing. It takes a list of the elements and information on them converts them into a more human readable form. It also makes it less of a hassle to get information on the elements when doing calculations for chemistry.

How does it work?

It take unformated JSON like below and converts it into a javascript object from which the selected pieces of information are input into a table, added to a div and, if it matches the search query, added to the DOM.
